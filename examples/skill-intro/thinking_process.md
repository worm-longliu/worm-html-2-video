## 阶段五：动画系统

### AI 的帧预算分配
```
帧率：30fps
场景淡入淡出：8帧（0.27s）

场景1(5s = 150帧)：
  淡入 8帧 → 内容 134帧 → 淡出 8帧
  元素1(气泡) delay=0 dur=0.5 (15帧动画)
  元素2(标题) delay=0.3 dur=0.5
  元素3(标签) delay=0.8 dur=0.5

场景2(7s = 210帧)：
  元素1(标题) delay=0 dur=0.5
  元素2(输入框) delay=0.4 dur=0.5
  元素3(分析卡片) delay=1.2 dur=0.5
  
...以此类推，确保每个元素在场景内有足够展示时间
```

### 动画原则
- 使用 easeOut 缓动函数（不用 linear，太机械）
- translateY(30px) 起始偏移（统一入场方向）
- 元素间 delay 递增 0.3-0.5s（节奏感）
- 避免同时出现太多动画（注意力分散）

## 总结：AI 的决策链路

```
接收指令
    ↓
需求分析（平台/受众/差异化）
    ↓
时长规划（查阅规范表 → 计算场景数）
    ↓
文案创作（语速验证 → 口语化 → TTS优化）
    ↓
视觉设计（CSS规范 → 安全区域 → 对比度）
    ↓
动画编排（帧预算 → 缓动函数 → 节奏控制）
    ↓
代码生成（HTML + JS帧驱动系统）
    ↓
自动化合成（截图 → 配音 → 字幕 → FFmpeg）
    ↓
质量验证（分辨率/音画同步/字幕检查）
    ↓
输出交付
```

**关键洞察**：AI 在每一步都参考了 skill 文档中的规范约束，确保输出质量。这就是"技能驱动"比"裸 prompt"效果好的原因 — 规范内化为 AI 的决策依据。
